### **Деньги**

В документе поступления на расчетный счет с видом операции «Поступление по платежным картам» обеспечен автоматический расчет и отражение комиссии банка при ручном распределении денежных средств, если включено отражение комиссии в настройках. Доначисление комиссии происходит в пропорциональном соотношении.

В загрузке операций по кошельку из Excel пустые строки не загружаются.

Добавлен новый документ «Перерасчет курса валют по кошельку». Документ предназначен для корректировки рублевых остатков по валютным кошелькам (счетам) в случае возникновения курсовых «хвостов» – когда остаток в валюте равен нулю, а в рублях остаются копейки, либо для принудительного пересчета рублевого эквивалента по заданному курсу.\
Инструмент позволяет привести в порядок валютный учет, закрыть незначительные остатки и обеспечить соответствие рублевых сумм текущему валютному остатку. Документ доступен в списке команд «Инструменты» раздела «Деньги».



### **Документы**

Для **1С:Управление нашей фирмой**  в документе «**Отчет комитенту**»  добавлены реквизиты P&L (дата принятия к управленческому учету, доп. аналитика, проект, раздел, статья, статья вознаграждения)  и формирование движений в регистр «Доходы».

Для **1С:Управление нашей фирмой** в документе «Отчёт о розничных продажах» добавлен реквизит P&L «Статья», если данный реквизит заполнен, то значение в движениях берется из него, если нет, то тогда значение берется из стандартного реквизита статьи.

Для **1С:Управление нашей фирмой**  в раздел «Документы» добавлен стандартный документ «Начисление налогов». В документе  добавлены реквизиты P&L (дата принятия к управленческому учету, доп. аналитика, проект, раздел, статья) и формирование движений в регистр «Расходы».

В  документе «Операция, введенная вручную» исправлена ошибка, когда при распределении не сохранялся «тип платежа», и не формировались движения в регистры P&L.

Для **1С:Управление торговлей** добавлено формирование движений в регистр «Движение денежных средств», если в разделе «Настройки», блок «Деньги» заполнена статья отклонения курса продажи валюты.

Для **1С:Управление нашей фирмой** по «Закрытию месяца» добавлено формирование движений по себестоимости в регистр «Расходы».



### **Управленческие документы**

В документе «Бюджет» добавлены две команды «Загрузить доходы» и «Загрузить расходы».

**Порядок действий:**

1. Нажмите нужную кнопку – откроется мастер загрузки данных.

2. Вставьте скопированные из Excel данные в специальное поле мастера.

3. Следуйте подсказкам мастера для сопоставления колонок и завершения загрузки.

Для всех управленческих документов отключена проверка на период закрытия.

В документе «Финансовый план (БДДС)»  при использовании команды «Заполнить по платежному календарю» заполнение происходит теперь только в тех колонках-месяцах, которые относятся к выбранному периоду.

Расширена функциональность учета кредитов и займов в части:

1. Добавлен выбор типа контрагента (юридическое / физическое лицо).

2. При редактировании в строке суммы основного долга происходит автоматический пересчет остатков.

3. Добавлена возможность создать платежный календарь по графику погашения, в этом случае на форме заполняются все реквизиты, кроме даты и суммы. Дата и сумма берутся из графика погашения.

4. Улучшен механизм загрузки и редактирования графиков платежей, а именно при нажатии команды «Загрузить график» пользователю предлагаются три варианта действий:

-  «Загрузить новый» – удаляет текущий график;

-  «Редактировать» – переносит текущий график во внешний табличный документ для редактирования пользователем;

-  «Отмена».

Для управленческого документа «Отражение заработной платы» реализовано создание на основании документа «Операция по кошельку».



### **Платежный календарь**

Добавлена возможность формировать график платежей ежедневно и еженедельно.

В отчете с установленной периодичностью «Группировка по дням» на текущую дату теперь отсутствует колонка «Просрочено».



### **Проекты**

В форме распределения документа с несколькими строками исправлена ошибка сохранения данных при детализации по номенклатуре.

В проект добавлен новый вариант отчета «Анализ расходов с отклонением».

Исправлены ошибки в отчетах проекта «Анализ доходов», «Анализ расходов» и «Бюджет + финансовый поток» по плановым суммам, накладным и деньгам. Перед проверкой обязательно нужно перезаписать проекты.

При режиме проектов «Профессиональный» добавлена возможность создания документа «Управленческая операция» с видом «Взаиморасчеты с контрагентами» по выбранному разделу проекта на основании данных сметы  (команда отображается при нажатии правой кнопки мыши по разделу). 



### **ДДС**

Во вкладке «Дашборд» для гистограммы «Общий денежный поток» была добавлена возможность группировки по видам деятельности и по доп. аналитике. Также добавлены новые блоки «По поступлениям» и «По списаниям». Для них реализована группировка по видам деятельности и по статьям.



### **ОПиУ**

Улучшена группа статьи ОПиУ, которая является формулой, а именно реализована возможность добавлять в формулу статьи движения денежных средств. Для этого в статье добавлен реквизит «Наименование для формулы». Если реквизит заполнен, то в форме редактирования формулы появится статья.

### **Соответствие статей и параметров**

В регистр «Соответствие параметров и статей ДДС» добавлена возможность указывать следующие реквизиты:

-  Проект;

-  Раздел проекта;

-  Дополнительная аналитика;

-  Отражать в незавершенном производстве.

Для **1С:Бухгалтерия** **предприятия** в документах «Поступление (акт, накладная, УПД)», «Реализация (акт, накладная, УПД)» и «Отчет о розничных продажах» изменена логика подбора значений реквизитов P&L с использованием соответствия параметров и статей ДДС.
